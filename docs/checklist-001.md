# Checklist 001 - Stock Analysis Agent Skill Discovery

1. Goal and desired artifact: identify Codex skills that are useful for building this project and write a project-local skill discovery index.
   -> Recommend and rank skills for the conversational stock-analysis AI agent; do not install skills yet.

2. Project domain: capture the domain vocabulary that should drive skill matching.
   -> Financial AI, Korean stocks, US stocks, equity research, market data, news ingestion, web search, web scraping, crawling, sentiment analysis, social listening, polling, LLM analysis, RAG, evidence scoring, explainability, backtesting, PnL charts, portfolio analytics, and ChatGPT-style UI.

3. Product lifecycle stage: clarify whether the request is ideation, design, implementation, testing, deployment, or review.
   -> Planning and implementation preparation; no application code should be generated by this request.

4. Target architecture: record the architecture direction that should influence skill selection.
   -> Incremental service-oriented architecture with Python backend, TypeScript frontend, PostgreSQL, source adapters, LLM provider adapters, scoring, market-data, and backtest services.

5. Primary implementation stack: record languages, frameworks, runtimes, and likely tooling.
   -> Python/FastAPI backend, TypeScript React or Next.js frontend, PostgreSQL, charting library to be selected later, and external APIs/crawlers for data collection.

6. Required source and integration areas: record external systems likely to need specialized guidance.
   -> OpenAI, Anthropic, Gemini, Naver news, Naver discussion boards, Reddit, global news, US sentiment or polling sources, Korean and US market data providers, EDGAR or public filings, search APIs, browser automation, crawler frameworks, vectorized backtesting tools, charting libraries, and secure key storage.

7. Safety and compliance constraints: record constraints that affect skill scoring.
   -> Treat the product as investment decision support, not automated trading or regulated financial advice; keep historical `as_of_at` evidence filtering strict; review crawler legality and API key handling.

8. Data and evaluation constraints: record special correctness requirements.
   -> Do not mix future evidence into historical LLM analysis; future price data may be used only for PnL/backtest views; source provenance and confidence must be auditable.

9. Maturity target: record the expected quality level.
   -> MVP moving toward production; recommendations should include high-confidence production skills plus broader candidates for later validation, but should label experimental or demo-oriented candidates clearly.

10. Search scope: record where candidates should be searched.
   -> Search installed project-local skills first, global/home-level skills as source candidates second, then external GitHub/community/official sources; prefer broad recall over narrow precision for this pass.

11. Project-local installation rule: record where selected skills must live if installed later.
   -> A skill counts as installed for this project only when `.codex/skills/<skill-name>/SKILL.md` exists.

12. Expected index path: record where the generated skill index should be written.
   -> `.find-skills/stock-analysis-agent/index.md`

13. Acceptance criteria: record how the skill discovery result should be judged.
   -> The index separates installed project-local skills, global source skills, and external candidates; ranks candidates by fit; includes precise, balanced, and recall buckets; explains why each skill matters; notes install status; and flags validation, license, portability, and dependency checks.

14. Exclusions: record what should not happen in this request.
   -> Do not install external skills, scaffold app code, create services, or change runtime dependencies until the generated index is confirmed.

15. Additional requests: preserve user-specific workflow requirements.
   -> Keep phase naming as `phase_00x`, keep AGENTS.md lightweight, preserve docs/task.md and docs/implement.md conventions, and keep backups for modified files.

16. Implementation workflow skills: include skills that help the project move from planning to implementation.
   -> Search for skills related to architecture, planning, task decomposition, TDD, systematic debugging, code review, API design, database design, migration planning, and validation.

17. Frontend and product UI skills: include skills that help build the user-facing application.
   -> Search for skills related to frontend design, React, Next.js, dashboard UI, chat UI, charting, data visualization, responsive layout, accessibility, and UX for analytical tools.

18. Backend and service skills: include skills that help build the service-oriented backend.
   -> Search for skills related to FastAPI, Python services, REST API design, async jobs, queues, background workers, scheduler workflows, repository structure, and modular service boundaries.

19. Data ingestion and crawler skills: include skills that help collect source evidence.
   -> Search for skills related to web scraping, crawler design, browser automation, robots/terms-aware collection, API ingestion, ETL pipelines, deduplication, source provenance, and metadata extraction.

20. LLM and agent skills: include skills that help make the assistant reliable.
   -> Search for skills related to prompt design, RAG, tool calling, provider abstraction, LLM evaluation, hallucination reduction, evidence-grounded answers, model comparison, and OpenAI/Claude/Gemini provider usage.

21. Financial analytics skills: include skills that help analyze stocks and price outcomes.
   -> Search for skills related to quant research, backtesting, VectorBT, yfinance, Korean market data, portfolio metrics, risk metrics, Sharpe/Sortino/drawdown, PnL simulation, and factor or sentiment analysis.

22. Security and privacy skills: include skills that reduce operational risk.
   -> Search for skills related to API key management, secret storage, encryption, prompt/data privacy, secure crawling, SSRF-safe URL ingestion, dependency audit, and threat modeling.

23. Testing and evaluation skills: include skills that validate behavior and model quality.
   -> Search for skills related to unit tests, integration tests, Playwright, API tests, golden datasets, LLM evals, historical cutoff tests, source filtering tests, scoring calibration, and regression testing.

24. Deployment and operations skills: include later-stage candidates even if they are not MVP-critical.
   -> Search for skills related to Docker, deployment, CI/CD, observability, logging, monitoring, job retries, data freshness checks, and hosted multi-user readiness.

25. Candidate breadth instruction: define how broad recommendations should be.
   -> Include direct-fit skills, adjacent workflow skills, and later-phase skills. Do not exclude a candidate only because it is not needed in `phase_002`; instead label its best phase and priority.
