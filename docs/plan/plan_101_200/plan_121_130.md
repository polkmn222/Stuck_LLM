Phase Goal
- Make scoring and PnL response contracts less ambiguous after the Claude review.

Completed Work
- Planned a confidence-adjusted downside probability that is separate from raw sell probability.
- Planned literal typing for `evaluation_kind: pnl_simulation` across backend and frontend boundaries.

Next Steps
- Treat future scoring metric changes as product-contract changes and update tests with the semantic definition.

Completed Work

Next Steps

## phase_123 - Chat Help And Malformed Provider Fallback

Phase Goal
- Make conversational stock workflows recoverable when users need command guidance or when a live provider returns malformed structured analysis output.

Completed Work
- Planned a local `/help` command that bypasses provider calls and returns a Korean or English guide inside the conversation.
- Planned malformed-output recovery that preserves `provider_error_code` while returning a local evidence fallback analysis for scoring.
- Planned a longer frontend request window and less misleading request-failure copy for slow news/provider requests.

Next Steps
- Keep fallback summaries clearly labeled so provider formatting failures do not look like clean live-provider analysis.

## phase_127 - Analysis And News Output Trust Cleanup

Phase Goal

## phase_128 - Malformed Output Provider Error Boundary

Phase Goal
- Remove malformed-output local prediction fallback so live stock predictions only complete when the LLM provider returns valid structured evidence.

Completed Work
- Planned conversion of malformed provider output from completed local fallback analysis to provider-error state.
- Planned removal of rule-based fallback scoring flags and user-facing fallback labels from active prediction paths.
- Planned product runtime documentation updates to state that malformed provider output must not synthesize local predictions.

Next Steps
- Treat future malformed provider responses as provider-quality issues to fix at the provider/schema layer, not as local prediction inputs.

## phase_129 - Multi Credential Selection Boundary

Phase Goal
- Let users save multiple LLM provider credentials and choose which encrypted key is used by the settings modal, chat header, and `/conversations` runtime.

Completed Work
- Planned a profile-based LLM credential API while preserving the existing single active `/credentials/llm` compatibility path.
- Planned per-message `llm_credential_id` metadata so chat intent, generic chat, news digest summarization, and live stock analysis use the same selected key.
- Planned frontend controls in both Settings and the chat workspace so model/key choice is visible before sending a request.

Next Steps
- Extend the same explicit credential-boundary thinking to external search/news/crawling provider keys in `phase_130`.

## phase_130 - Crawl-Backed News And Scenario Prediction

Phase Goal
- Make ChatGPT-style Apple news and prediction requests use real provider headlines, user-supplied crawlable URLs, Reddit search results when available, and source-grounded scenario analysis.

Completed Work
- Planned a safe direct URL crawl path for user-supplied article URLs with scheme, host, credential, redirect, and response-size boundaries.
- Planned Reddit search URL handling through Reddit search JSON when a public search URL is supplied.
- Planned conversion of news digest articles into dated source documents before live prediction prompts, preserving the existing `as_of_at` cutoff.
- Planned scenario-formatted prediction copy that keeps buy/hold/sell probabilities while adding base, bull, and bear cases.

Next Steps
- Keep blocked crawl attempts visible through provider-run warnings instead of silently treating missing pages as neutral evidence.
