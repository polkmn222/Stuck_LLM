# Plan Phase 131-140 Summary

This file is a compact planning index for agents. Keep the detailed source log separate when it exists.

## phase_131 - Free RSS News Providers

Phase Goal
- Add no-key news discovery providers so ChatGPT-style headline requests can return real linked articles without requiring paid search keys.

Completed Work
- Planned `seekingalpha_rss`, `yahoo_finance_rss`, `google_news_rss`, and `bing_news_rss` as free providers connected to existing ranking, dedupe, provider-run, and cache boundaries.

Next Steps
- Keep RSS parsing defensive and provider metadata explicit because feed contents are not trusted evidence.

## phase_132 - EventRegistry News Provider

Phase Goal
- Add EventRegistry as an optional premium news provider behind the external credential boundary.

Completed Work
- Planned direct REST integration instead of adding an SDK dependency.
- Planned credential resolution so selected EventRegistry keys are used before any local-development environment fallback.

Next Steps
- Treat future EventRegistry behavior changes as provider-contract changes with credential and raw-secret tests.

## phase_133 - Reddit Public Search Provider

Phase Goal
- Add Reddit public search as a social/community source path without mixing it into normal article-provider semantics.

Completed Work
- Planned subreddit search fanout for community, Reddit, and sentiment-style requests.
- Kept user-supplied Reddit search URL crawl behavior separate from the automatic public-search provider.

Next Steps
- Keep Reddit output as untrusted source metadata until converted into source documents and filtered by `as_of_at`.

## phase_134 - ChatGPT-Style News And Scenario Replies

Phase Goal
- Make news and prediction replies look closer to the user's target ChatGPT format while preserving source grounding.

Completed Work
- Planned sectioned news digest copy with actual headline links, source/provider labels, publication dates, and `as_of_at`.
- Planned information-based base, bull, and bear scenario prediction wording without removing buy, hold, sell probabilities.

Next Steps
- Keep the response format auditable; do not replace source links with unsupported prose summaries.

## phase_135 - External News Provider Credential Selection

Phase Goal
- Let user-selected external/news API keys drive news providers, independent from selected LLM keys.

Completed Work
- Planned encrypted external credential profiles for Tavily, GNews, SerpAPI, and EventRegistry.
- Planned conversation runtime resolution so selected saved external keys are passed into news discovery and prediction news context.

Next Steps
- Future market-data or crawler-specific keys should use the same selected-credential boundary instead of reusing LLM keys.

## phase_136 - Frontend Credential And Source Badges

Phase Goal
- Expose the new provider/key behavior in Settings and Chat so users can see which key and sources are in play.

Completed Work
- Planned Settings UI for external/news API keys, Chat selected credential badges, clickable markdown source links, and separate source/provider badges in news cards.

Next Steps
- Keep credential UI masked and provider-specific, and avoid showing raw API keys or hidden prompts in component state snapshots.

## phase_137 - Product And Runtime Documentation Update

Phase Goal
- Update product/runtime docs and compact summaries after phases 131-136 changed provider, credential, response-format, and UI behavior.

Completed Work
- Planned updates to `llm-agent-spec.md`, `llm-runtime-execution.md`, and the phase 131-140 compact indexes.

Next Steps
- Keep this range summary current if phases 138-140 add more behavior before the next compact bucket.

## phase_138 - Reddit Sentiment Query Boundary

Phase Goal
- Tighten Reddit public search routing so sentiment/social-reaction requests are treated as community-source requests and retain user topical context.

Completed Work
- Planned a focused news-digest service change: expand Reddit activation keywords and build Reddit queries from company/ticker plus the cleaned user request.

Next Steps
- Preserve the existing X/social `SOCIAL_QUERY_SUFFIX`; Trump/tariffs terms remain available for social-web searches when SerpApi social search is selected.

## phase_139 - Reddit Partial Failure Visibility

Phase Goal
- Make partial Reddit subreddit failures auditable while keeping successful public-search articles in the digest.

Completed Work
- Planned a soft-warning path for partial Reddit fan-out failures instead of changing the provider-run status enum.

Next Steps
- Keep the all-failed path as `provider_error`; only mixed success/failure should retain articles with a warning.

## phase_140 - EventRegistry Payload Guard

Phase Goal
- Prevent malformed EventRegistry responses from bypassing provider-error handling.

Completed Work
- Planned strict validation for the `articles.results` payload boundary and a type tightening for selected external credential maps.

Next Steps
- Keep the EventRegistry host unchanged unless live API verification shows a real redirect or endpoint incompatibility.
