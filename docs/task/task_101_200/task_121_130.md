# Task Phase 121-130 Summary

This file is a compact task index for agents. Keep the detailed source log separate when it exists.

## phase_121 - Scoring And PnL Contract Cleanup

Phase Goal
- Tighten scoring and PnL response semantics identified in the Claude review.

Acceptance Criteria
- `downside_probability` is no longer a copy of `sell_probability`.
- The scoring rationale explains how downside probability differs from sell probability.
- Backend `BacktestResponse.evaluation_kind` accepts only `pnl_simulation`.
- Frontend API/types preserve `evaluationKind` as a required literal.

Completed Work
- Added scoring/PnL contract tests.
- Implemented confidence-adjusted downside probability.
- Narrowed backend and frontend PnL evaluation-kind types.

## phase_122 - Analysis And News UI Label Cleanup

Phase Goal

Acceptance Criteria

- Confidence factors render as localized user-facing labels.
- Analysis provider-state aria-label is localized.
- Provider-run keys remain stable when provider/query pairs repeat.

Completed Work

## phase_123 - Chat Help And Malformed Provider Fallback

Phase Goal
- Improve the chat path reported by users for `/help`, Apple prediction requests, and slow news requests.

Acceptance Criteria
- `/help` returns a localized capability guide without requiring credentials or calling the LLM provider.
- Malformed live-provider analysis output does not block predictions when eligible evidence exists.
- Malformed-output fallback responses keep `provider_error_code=malformed_output` visible and use only local eligible evidence for scoring.
- Chat request failures no longer say the message could not be saved locally when the request may have timed out or failed upstream.
- Conversation/news requests have a longer client timeout window.

Completed Work
- Added failing backend tests for `/help` and malformed-output fallback.
- Added failing frontend tests for the longer timeout window and Korean request-failure copy.
- Implemented the local help command, malformed-output local evidence fallback, and frontend request/copy adjustments.

## phase_127 - Analysis And News Output Trust Cleanup

Phase Goal
- Fix output-quality defects reported for Apple prediction and Apple news flows.

- Rule-based malformed-output fallback scoring caps confidence at mid-confidence and includes an explicit `llm_unavailable` confidence factor.

## phase_128 - Malformed Output Provider Error Boundary

Phase Goal
- Ensure stock prediction never completes through local fallback when the LLM provider returns malformed structured output.

Acceptance Criteria
- `malformed_output` from the live analysis provider returns conversation `status=provider_error`.
- Malformed-output responses contain no local evidence items, no score result, no probability text, and no fallback copy.
- Source audit state still preserves included and excluded source decisions for troubleshooting.
- Scoring no longer exposes `rule_based_fallback` or `llm_unavailable` fallback metadata.
- Frontend provider-error labels do not imply a rule-based fallback.
- Product runtime docs state that malformed provider output must not synthesize local predictions.

Completed Work
- Added failing backend tests for malformed-output provider-error behavior.
- Updated frontend provider-error expectations to remove fallback wording.
- Removed malformed-output local prediction fallback and related scoring/UI metadata.

Completed Work

- Added frontend API, Settings modal, App, and ChatShell tests for profile mapping and key selection.
- Implemented profile endpoints and per-message credential selection.

## phase_130 - Crawl-Backed News And Scenario Prediction

Phase Goal
- Improve Apple-style news and prediction output so it uses actual headlines and source-backed scenario analysis.

Acceptance Criteria
- News digest can include a crawlable article URL, such as a Seeking Alpha article URL, as a source-linked article when fetching succeeds.
- News digest can convert a Reddit search URL into Reddit result articles when the public JSON endpoint is available.
- Crawled or provider news articles can become stock-analysis source documents only after `as_of_at` filtering.
- Prediction assistant text includes information-based base, bull, and bear scenarios while preserving buy/hold/sell probabilities.
- If crawling fails or is blocked, provider-run warnings remain visible instead of fabricating article content.

Completed Work
- Added tests for direct URL crawl, Reddit search crawl, news-to-prediction source conversion, and Korean scenario prediction copy.
- Implemented direct web and Reddit crawl provider runs in the news digest service.
- Converted important news digest articles into source documents for live prediction prompts.
- Updated prediction message formatting with base, bull, and bear scenario sections.
