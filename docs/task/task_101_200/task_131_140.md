# Task Phase 131-140 Summary

This file is a compact task index for agents. Keep the detailed source log separate when it exists.

## phase_131 - Free RSS News Providers

Phase Goal
- Add free RSS news providers to the existing news digest system.

Acceptance Criteria
- Seeking Alpha, Yahoo Finance, Google News, and Bing News RSS providers are available provider values.
- Free RSS providers feed existing provider runs, ranking, URL dedupe, and cache behavior.
- Chat news requests can return actual linked headlines without paid provider keys.

Completed Work
- Added provider schema values, RSS fetch/parsing logic, default no-key provider expansion, and focused backend tests.

## phase_132 - EventRegistry News Provider

Phase Goal
- Add EventRegistry as a premium credential-based news provider.

Acceptance Criteria
- EventRegistry has an external provider credential definition.
- EventRegistry news runs use explicit selected keys when provided.
- Provider runs preserve missing-credential and provider-error states without exposing raw keys.

Completed Work
- Added EventRegistry credential metadata, news provider schema support, REST provider logic, and focused tests.

## phase_133 - Reddit Public Search Provider

Phase Goal
- Add automatic Reddit public search for community/social news requests.

Acceptance Criteria
- Reddit public search activates only for Reddit, community, investor sentiment, or similar terms.
- Subreddit search results normalize into news digest articles with provider transparency.
- Existing user-supplied Reddit URL crawl remains supported.

Completed Work
- Added Reddit public search provider value, subreddit search logic, request detection, dedupe, and focused tests.

## phase_134 - ChatGPT-Style News And Scenario Replies

Phase Goal
- Improve user-facing news and prediction formatting while keeping evidence visible.

Acceptance Criteria
- News digest replies include `as_of_at`, actual headline links, source/provider labels, and categorized sections.
- Prediction replies preserve local scoring output and present base, bull, and bear cases as information-based scenarios.
- Korean and English copy remain language-aware.

Completed Work
- Updated conversation response formatting and added tests for source-linked sectioned news and scenario-style analysis copy.

## phase_135 - External News Provider Credential Selection

Phase Goal
- Make external/news provider keys user-selectable and separate from LLM credentials.

Acceptance Criteria
- Settings/runtime can save, list, select, and delete encrypted external provider credential profiles.
- Active external credentials are passed into news digest and prediction news context.
- Selected keys override local environment compatibility keys.
- Raw keys are absent from state and API responses.

Completed Work
- Added backend state, schemas, service functions, credential endpoints, runtime wiring, and focused credential-selection tests.

## phase_138 - Reddit Sentiment Query Boundary

Phase Goal
- Make Reddit public search honor investor-sentiment and social-reaction requests without dropping the user's topical terms.

Acceptance Criteria
- Reddit public search activates for investor sentiment, social reaction, and Korean sentiment terms, not only explicit Reddit/community/forum terms.
- Reddit public search queries include the resolved company/ticker and the cleaned user request so topical terms such as Trump, tariffs, or X/social reaction are preserved.
- Existing social-web query behavior, including the Trump/tariffs suffix used for X/social search, remains unchanged.

Completed Work
- Added sentiment/social-reaction activation coverage for Reddit public search and preserved user topical terms in Reddit provider queries.

## phase_139 - Reddit Partial Failure Visibility

Phase Goal
- Keep Reddit public search best-effort, but record partial subreddit fan-out failures instead of marking them fully invisible.

Acceptance Criteria
- If at least one Reddit subreddit request succeeds and at least one fails, the digest keeps returned articles and records a provider warning.
- The provider run remains user-safe and does not drop successful Reddit articles.
- Existing all-failed Reddit behavior still surfaces as a provider error.

Completed Work
- Added a soft-warning path for partial Reddit subreddit failures while preserving successful Reddit articles.

## phase_140 - EventRegistry Payload Guard

Phase Goal
- Harden EventRegistry response handling and tighten external credential map typing.

Acceptance Criteria
- Malformed EventRegistry payloads are recorded as provider errors instead of crashing digest generation.
- Empty but well-formed EventRegistry result lists remain valid empty results.
- External credential maps use the external provider literal key type rather than arbitrary strings.

Completed Work
- Added EventRegistry payload validation so malformed `articles.results` values become provider errors, and tightened external credential map typing.
